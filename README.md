# 音乐服务器管理后台

一个简易的音乐库与歌单管理面板，提供 Meting API 格式请求。适用于个人音乐服务器、网页播放器或移动客户端的后台数据管理。

## 主要功能

- 歌单管理：支持多歌单切换、曲目增删改查、批量删除、拖拽排序（支持多选拖拽）、重复项处理（覆盖/合并/取消）。
- 智能导入：自动扫描本地媒体目录，解析文件名提取曲名与艺术家，自动匹配同名 LRC 歌词与封面图片。
- 文件管理：浏览、上传、重命名、删除本地媒体文件。支持拖拽上传、实时进度显示、批量勾选下载（自动打包）。
- 备份与恢复：支持歌单数据版本化备份。
- API 生成：每个歌单均使用标准 JSON 存储，方便外部播放器或第三方应用直接调用。
- 响应式设计：采用 Bootstrap 5 构建，适配桌面与移动端。

## 技术栈

- 后端：PHP 7.4+（需启用 ZipArchive 与 JSON 扩展）
- 前端：HTML5，CSS3，JavaScript
- UI 框架：Bootstrap 5.3.3
- 数据格式：JSON
- 依赖：无第三方 Composer/NPM 依赖，开箱即用

## 环境要求

- PHP 7.4 或更高版本
- Web 服务器：Apache（推荐 2.4+） \/ Nginx \/ IIS 等
- PHP 扩展：`zip`, `json`, `session`（默认已包含）
- 浏览器：支持现代 Web 标准的主流浏览器（Chrome, Firefox, Safari, Edge 等）

## 安装与部署

1. 将项目文件完整上传至 Web 服务器的可访问目录。
2. 确保 PHP 环境已启用 `ZipArchive` 扩展（用于批量下载打包）。
3. 通过浏览器访问 `admin.php`。系统首次运行时会自动创建 `playlists/`、`localmedia/` 及 `playlists/backups/` 目录。
4. 默认登录凭据：用户名 `admin`，密码 `123456`。
5. 部署至生产环境前，请务必修改 `admin.php` 顶部的 `ADMIN_PASS_HASH` 量。

## 使用说明

### 歌单管理
- 使用顶部下拉菜单切换或创建、重命名、删除歌单。
- 点击“添加曲目”手动录入，或使用“智能添加”自动扫描媒体目录并批量导入。
- 进入“顺序编辑”模式后，可拖拽手柄调整曲目位置。勾选多个曲目后拖拽可批量移动。
- 数据修改仅在内存中生效，需点击“保存”才会写入服务器 JSON 文件。
- 展开底部“高级 JSON 编辑”可直接修改原始数据，改动会实时同步至表格。

### 文件管理
- 通过面包屑导航浏览媒体目录结构。
- 支持拖拽文件至上传区域，或点击按钮选择文件。上传过程显示实时进度条。
- 勾选文件或文件夹后，可执行批量删除或批量下载（服务端打包为 ZIP 返回）。

### 系统设置与备份
- 在“系统设置”中配置歌单与媒体目录路径、默认元数据（曲名、艺术家、封面、歌词路径）及文件名解析规则（分隔符与格式顺序）。
- “备份管理”支持按时间戳存档当前歌单状态，可随时恢复历史版本或下载备份文件。

### API 接口
- 访问“API 预览”页面可查看当前所有歌单对应的 JSON 接口地址。
- 接口返回标准数组格式，包含 `name`, `artist`, `url`, `lrc`, `pic` 字段，可直接用于前端播放器渲染。

## 安全与配置说明

- 身份验证：修改默认密码需使用 PHP 的 `password_hash()` 生成新哈希值，并替换 `admin.php` 中的 `ADMIN_PASS_HASH`。
- 路径安全：所有文件操作均通过 `fm_sanitize_path()` 进行严格校验，限制在配置的根目录内，防止目录遍历攻击。
- 生产建议：务必通过 HTTPS 协议访问管理后台。建议通过 Web 服务器配置（如 `.htaccess` 或 Nginx 访问控制）限制 IP 访问范围，避免暴露在公网。
